# TCP Application Example: Open Multi-Layer Argument Audit Framework

**A practical application example of the Three-Term Closure Principle (TCP)**  
**Version:** v0.1-public  
**Status:** Application framework example, not a core specification  
**Language order:** English first / Japanese reference second

---

## Status of This Document

This document reformats and positions the **Open Multi-Layer Argument Audit Framework** as a practical application example of TCP.

It is not a new core theory.  
It is not evidence that TCP is true.  
It is not a substitute for the TCP specification.

Its role is narrower:

```text
To show how TCP-style closure can be applied to natural-language argument audit.
```

In this document, argument audit is treated not as a static evaluation of isolated propositions, but as a multi-layer inspection of the conditions that make an argument possible.

---

## Relationship to TCP

TCP defines a world-description unit as:

```text
W := (X, R, M)
```

where:

- `X` = object / target / scope
- `R` = relation / observation / comparison / transition
- `M` = closure function / identity / boundary / judgment / stopping rule

This application framework maps TCP into argument audit as follows:

| TCP element | Argument-audit interpretation | Main layer |
|---|---|---|
| `X` | What claim, subject, object, domain, time, space, and speaker are being audited | Layer 0 |
| `R` | How the claim relates, transitions, updates, branches, and derives conclusions | Layer 1 / Layer 3 |
| `M` | Definitions, premises, scope, uncertainty, judgment standards, stopping rules | Layer 2 / Layer 4 |
| `SUSPEND` | Valid halt state when coordinates, dynamics, or tacit elements are unclosed | Layer 5 |

The main TCP insight applied here is:

```text
A claim cannot be audited as closed until its object, relation, and closure rules are fixed.
```

---

## Inviolable Boundary

This framework must not be used for:

1. superiority scoring of persons, groups, or models;
2. coercive persuasion or manipulation;
3. authority-based silencing;
4. pretending that an unclosed argument is settled;
5. converting argument audit into ideological enforcement.

The purpose is not to “win arguments.”  
The purpose is to detect whether an argument is sufficiently closed to be evaluated.

---

## Abstract

This framework reconstructs argument audit as an open multi-layer process. Conventional argument evaluation often begins at the explicit proposition layer: validity, coherence, and adequacy. However, many arguments collapse before that layer becomes meaningful.

They collapse because:

- the coordinates are unfixed;
- the dynamic conditions are omitted;
- tacit definitions, premises, scope, and uncertainty remain hidden;
- consistency is asserted without cross-layer comparison;
- reserved or unapplied audit items are mistaken for nonexistence.

Therefore, this document treats argument audit as a six-layer process:

```text
Layer 0: Coordinate Anchoring
Layer 1: Dynamic Transition
Layer 2: Tacit-Knowledge Audit
Layer 3: Explicit Argument Audit
Layer 4: Integrated Judgment
Layer 5: Reservation / SUSPEND Layer
```

The framework is open-ended. It does not claim that these six layers exhaust all possible audit items. Instead, it provides a stable minimal structure for detecting common failure modes in natural-language arguments.

## 1. Purpose

The purpose of this framework is to audit natural-language arguments by asking:

```text
Is this claim sufficiently closed to be evaluated?
```

It does not first ask whether the claim is true.  
It first asks whether the claim has fixed enough coordinates, relations, and judgment rules to become auditable.

---

## 2. Primary Target

This framework primarily targets natural-language argumentation, including:

- public statements;
- policy claims;
- institutional claims;
- technology and AI discourse;
- explanatory documents;
- recommendation documents;
- essays;
- AI-generated evaluations;
- research claims expressed in natural language.

It is not primarily designed for:

- fully formal mathematical proofs;
- completed symbolic logic systems;
- purely factual lists without argumentative structure;
- aesthetic or poetic expressions as such.

Some layers may still apply supplementarily to those cases, but they are not the primary target.

---

## 3. Core Hypothesis

The framework adopts the following hypothesis:

```text
Many argument failures occur before the explicit proposition layer.
```

In other words, an argument may appear valid, coherent, or adequate at the surface level while still failing because:

- the speaker is unfixed;
- the acting agent is unfixed;
- the target domain is unfixed;
- the temporal condition is unfixed;
- the mechanism is unfixed;
- definitions shift;
- premises are silently imported;
- scope expands or contracts opportunistically;
- uncertainty is disguised as certainty.

Therefore, argument audit must be multi-layered.

---

## 4. Layer 0: Coordinate Anchoring

### 4.1 Function

Layer 0 fixes the coordinates of the argument before content-level audit begins.

At minimum, the following must be fixed:

```text
Who
What
When
Where
Why
How
```

In this framework, 5W1H is not treated as a writing template.  
It is treated as a coordinate system.

### 4.2 Audit Questions

- Who is speaking?
- Who is the acting agent?
- What is the object of change or judgment?
- When does the claim apply?
- Where does it apply?
- Why is the claim being made?
- How is the claimed result supposed to occur?

### 4.3 TCP Mapping

```text
Layer 0 mainly fixes X.
```

Without Layer 0, the target of audit is unstable.

---

## 5. Layer 1: Dynamic Transition

### 5.1 Function

Arguments often describe or imply transitions. They are rarely still images.

Layer 1 audits the dynamic structure behind the claim.

### 5.2 Minimum Items

- initial state
- input
- processing rule
- intermediate state
- output
- branching condition
- update condition
- stopping condition
- feedback loop

### 5.3 Typical Failure Modes

- only the conclusion is shown while the process is black-boxed;
- a temporary condition is treated as a permanent rule;
- branching paths are erased;
- updates are ignored;
- feedback effects are omitted.

### 5.4 TCP Mapping

```text
Layer 1 mainly fixes R.
```

It clarifies how states, relations, and transitions are assumed to operate.

---

## 6. Layer 2: Tacit-Knowledge Audit

### 6.1 Function

Layer 2 extracts hidden supports behind the argument.

This framework uses four core audit items:

```text
definition
premise
scope
uncertainty
```

These are not exhaustive. They are the minimum core items adopted for this application framework.

### 6.2 Definition

Audit questions:

- Are key terms explicitly defined?
- Did the definition shift midway?
- Was a definition silently imported from another context?

### 6.3 Premise

Audit questions:

- What must be assumed for the claim to hold?
- Are the premises factual, institutional, value-based, comparative, or observational?
- Are necessary premises and auxiliary premises distinguished?

### 6.4 Scope

Audit questions:

- What is the valid range of the claim?
- Is it universal or condition-bound?
- Is it domain-specific?
- Are non-applicability conditions stated?

### 6.5 Uncertainty

Audit questions:

- Is the claim fact, inference, hypothesis, prediction, impression, or wish?
- Does the assertion strength match the support strength?
- Is uncertainty being disguised as certainty?

### 6.6 TCP Mapping

```text
Layer 2 mainly fixes M.
```

It provides the closure conditions needed to interpret and judge the argument.

---

## 7. Layer 3: Explicit Argument Audit

### 7.1 Function

Only after the prior layers are fixed does the framework evaluate explicit argument quality.

Layer 3 uses three items:

```text
validity
coherence
adequacy
```

### 7.2 Validity

Does the inference form hold?  
Are there inferential leaps, circular moves, or unsupported conclusions?

### 7.3 Coherence

Do the statements fit together within the given context?

Coherence is contextual.  
It is not merely the absence of contradiction in isolated sentences.

### 7.4 Adequacy

Are the premises, grounds, observations, causal assumptions, and comparisons adequate to the strength of the conclusion?

A valid argument may still fail if the premises are weak or rotten.

### 7.5 TCP Mapping

```text
Layer 3 audits R under the M conditions extracted in Layer 2.
```

---

## 8. Layer 4: Integrated Judgment

### 8.1 Function

Layer 4 performs cross-layer comparison.

It asks whether the argument remains stable after crossing:

- coordinates;
- dynamics;
- tacit knowledge;
- explicit proposition structure.

### 8.2 Consistency

This framework distinguishes coherence and consistency.

```text
Coherence = local fit within a given context.
Consistency = stability of governing principles across layers and contexts.
```

### 8.3 Failure Types

Typical consistency failures include:

- definition-shift type;
- premise-substitution type;
- scope-stretching or shrinking type;
- uncertainty-disguise type;
- unauthorized coordinate-shift type;
- dynamic-omission type.

### 8.4 TCP Mapping

```text
Layer 4 checks whether W remains stable across X / R / M.
```

---

## 9. Layer 5: Reservation / SUSPEND Layer

### 9.1 Function

Layer 5 prevents the framework from pretending to be complete.

It holds:

- items acknowledged but not unfolded;
- items not yet operationalized;
- items outside the current scope;
- items that would diffuse the core structure;
- future expansion candidates.

### 9.2 SUSPEND

When coordinates, dynamics, tacit supports, or judgment rules remain unclosed, the correct output is not over-assertion.

The correct output is:

```text
SUSPEND
```

### 9.3 TCP Mapping

```text
Layer 5 is the operational expression of TCP-style stoppability.
```

---

## 10. Standard Audit Procedure

1. **Coordinate anchoring**  
   Fix Who / What / When / Where / Why / How.

2. **Dynamic extraction**  
   Extract initial state, input, transition, branching, updating, stopping, and feedback.

3. **Tacit-knowledge extraction**  
   Extract definition, premise, scope, and uncertainty.

4. **Tacit-knowledge assessment**  
   Check whether the extracted tacit elements fit the coordinates and dynamics.

5. **Explicit argument assessment**  
   Check validity, coherence, and adequacy.

6. **Integrated judgment**  
   Compare across layers and identify consistency failures.

7. **Reservation disclosure**  
   State what remains outside the current audit and whether SUSPEND is required.

---

## 11. Example: “AI Will Change the World”

The sentence:

```text
AI will change the world.
```

may be meaningful as a slogan, but it is not yet an auditable claim.

### 11.1 Coordinate Failures

- Who is speaking?
- What does “AI” mean?
- What does “the world” mean?
- When will the change occur?
- Where will it occur?
- Why does the claim matter?
- How does the change happen?

### 11.2 Agent Ambiguity

The acting agent is unclear:

- Does AI change itself?
- Does AI directly change something?
- Do humans change behavior through AI use?
- Do institutions change because they adopt AI?
- Do markets change because AI alters incentives?

### 11.3 Scope Ambiguity

“The world” may mean:

- technology;
- economy;
- social institutions;
- human cognition;
- relationships;
- power structures;
- information environment.

These cannot be audited as one undifferentiated object.

### 11.4 Dynamic Ambiguity

The mechanism is missing:

- adoption;
- use;
- operation;
- institutional change;
- behavioral change;
- power shift;
- feedback;
- regulation;
- failure and correction.

### 11.5 Audit Result

```text
Decision: SUSPEND
Reason: Coordinate and dynamic conditions are not sufficiently fixed.
```

The sentence can function as rhetoric, but it does not yet function as an auditable proposition.

---

## 12. Example: Valid Form, Failed Audit

Claim:

```text
This system increases efficiency.
Systems that increase efficiency are desirable.
Therefore, this system is desirable.
```

The form may appear valid.

However, the audit fails if the following are not fixed:

- efficient for whom?
- efficient for what task?
- efficient over what time horizon?
- efficient in what institutional space?
- what side effects are included?
- what value premise makes efficiency desirable?
- is the definition of efficiency stable?
- what is the scope?
- how is uncertainty handled?

Possible result:

```text
Decision: SUSPEND or FAIL
Reason: Tacit premises and scope are underdefined.
```

---

## 13. Example: Local Coherence Without Consistency

Context A:

```text
A probabilistic output system should not be given final decision authority.
```

Context B:

```text
This probabilistic system is high-performing, so final decision authority may be entrusted to it.
```

Each context may be locally coherent.

But the cross-layer judgment principle is unstable unless the framework explains why performance overrides probabilisticity in Context B.

Possible result:

```text
Decision: FAIL or SUSPEND
Reason: Inconsistent governing principle across contexts.
```

---

## 14. Short Checklist

1. Who is the speaking subject?
2. Who is the acting agent?
3. What is the object of change or judgment?
4. What is the temporal condition?
5. What is the spatial condition?
6. What is the dynamic condition?
7. Are the definitions fixed?
8. Have the premises been extracted?
9. Is the scope delimited?
10. Is uncertainty distinguished?
11. Is the argument valid?
12. Is it coherent?
13. Is it adequate?
14. Is it consistent across layers?
15. What remains reserved?
16. Should the result be PASS, SUSPEND, or FAIL?

---

## 15. Significance

This framework is useful because it:

1. extends argument audit from propositions to conditions of existence;
2. introduces time, space, and dynamics into audit;
3. separates coherence from consistency;
4. makes tacit knowledge extractable;
5. prevents compressed models from pretending to be closed totalities;
6. gives TCP a practical audit use case.

---

## 16. Limitations

### 16.1 Auditor Dependence

Coordinate anchoring and tacit-knowledge extraction require auditor competence.

### 16.2 Operationalization Gap

Future work should convert the layers into:

- questionnaires;
- checklists;
- reason codes;
- decision tables;
- test cases.

### 16.3 Empirical Status

This is a conceptual and methodological framework.  
It is not yet an empirically validated audit benchmark.

### 16.4 Self-Scope Limitation

This framework is itself open.  
It should not be treated as a completed total theory.

# TCP適用例：開放型・多層論証監査フレームワーク

**トリニティ原理（TCP）の実用適用例**  
**版：** v0.1-public  
**位置づけ：** 中核仕様ではなく、適用フレームワーク例  
**言語順：** 英語先頭 / 日本語参考訳後段

---

## 本文書の位置づけ

本文書は、**開放型・多層論証監査フレームワーク**を、TCPの実用適用例として再整形したものである。

これは新しい中核理論ではない。  
TCPの証明でもない。  
TCP仕様書の代替でもない。

役割は限定される。

```text
TCP型の閉包を、自然言語の論証監査へどう適用できるかを示す。
```

本文書では、論証監査を「表面上の命題の静的評価」ではなく、論証を成立させている条件群の多層検査として扱う。

---

## TCPとの関係

TCPは、世界記述単位を次で定義する。

```text
W := (X, R, M)
```

ここで：

- `X` = 対象 / ターゲット / 射程
- `R` = 関係 / 観測 / 比較 / 遷移
- `M` = 閉包機能 / 同一性 / 境界 / 判定 / 停止規則

本適用フレームワークでは、これを論証監査に次のように写像する。

| TCP要素 | 論証監査での解釈 | 主な層 |
|---|---|---|
| `X` | どの主張・主体・対象・領域・時空間・話者を監査するか | Layer 0 |
| `R` | 主張がどう関係し、遷移し、更新され、分岐し、結論へ至るか | Layer 1 / Layer 3 |
| `M` | 定義・前提・射程・不確実性・判定基準・停止規則 | Layer 2 / Layer 4 |
| `SUSPEND` | 座標・動態・暗黙要素が未閉包である場合の正当な停止出力 | Layer 5 |

ここで適用されるTCPの要点は次である。

```text
対象・関係・閉包規則が固定されるまで、主張は閉包済みとして監査できない。
```

---

## 不可侵境界

本フレームワークを以下に用いてはならない。

1. 個人・集団・モデルの優劣スコアリング
2. 強制的説得・操作・誘導
3. 権威による沈黙強制
4. 未閉包の論証を確定済みと偽装する行為
5. 論証監査を思想統制へ変換する行為

目的は「議論に勝つこと」ではない。  
目的は、その論証が評価可能な程度に閉じているかを検出することである。

---

## 要旨

本フレームワークは、論証監査を開放型の多層プロセスとして再構成する。従来の論証評価は、妥当性・整合性・十分性といった明示命題層から始まりやすい。しかし多くの論証は、その層に到達する前に崩れている。

崩れる理由は次である。

- 座標が固定されていない
- 動態条件が省略されている
- 定義・前提・射程・不確実性が暗黙化されている
- 一貫性が層横断比較なしに主張されている
- 予約層に置くべき未展開項目が、存在しないものとして扱われている

したがって、本文書は論証監査を次の6層で扱う。

```text
Layer 0: 座標固定層
Layer 1: 動態遷移層
Layer 2: 暗黙知監査層
Layer 3: 明示論証監査層
Layer 4: 統合判定層
Layer 5: 予約 / SUSPEND層
```

本フレームワークは開放系である。  
これら6層がすべての監査項目を尽くすとは主張しない。  
自然言語論証における典型的な失敗モードを検出するための、安定した最小構造を提示する。

## キーワード

論証監査、TCP適用、三項閉包、座標固定、動態遷移、暗黙知、定義、前提、射程、不確実性、妥当性、整合性、十分性、一貫性、予約層、SUSPEND

---

## 1. 目的

本フレームワークの目的は、自然言語の論証に対して次を問うことである。

```text
この主張は、評価可能な程度に閉じているか。
```

最初に問うのは真偽ではない。  
まず、その主張が監査可能になるだけの座標・関係・判定規則を固定しているかを問う。

---

## 2. 主対象

本フレームワークの主対象は、自然言語ベースの論証である。

例：

- 公的発言
- 政策主張
- 制度に関する主張
- 技術・AIをめぐる言説
- 説明文書
- 提案文書
- エッセイ
- AI生成の評価文
- 自然言語で提示された研究主張

主対象ではないもの：

- 完全に形式化された数学的証明
- 閉じた記号論理体系
- 論証構造を持たない事実リスト
- 美的・詩的表現そのもの

ただし、一部の層は補助的に適用できる可能性がある。

---

## 3. 核心仮説

本フレームワークは、次の仮説を採用する。

```text
多くの論証破綻は、明示命題層より前で発生する。
```

すなわち、表面上は妥当・整合・十分に見える論証であっても、以下により崩れる。

- 話者が固定されていない
- 行為主体が固定されていない
- 対象領域が固定されていない
- 時間条件が固定されていない
- メカニズムが固定されていない
- 定義がずれる
- 前提が暗黙輸入される
- 射程が都合よく拡大・縮小される
- 不確実性が確実性に偽装される

ゆえに、論証監査は多層構造でなければならない。

---

## 4. Layer 0：座標固定層

### 4.1 機能

Layer 0は、内容監査に入る前に論証の座標を固定する。

最低限、次を固定する。

```text
Who
What
When
Where
Why
How
```

本フレームワークでは、5W1Hを文章作成テンプレートではなく、座標系として扱う。

### 4.2 監査質問

- 誰が語っているのか
- 誰が行為主体なのか
- 何が変化・判定の対象なのか
- いつその主張は成立するのか
- どこで成立するのか
- なぜその主張がなされているのか
- どのように結果が生じるのか

### 4.3 TCP写像

```text
Layer 0は主にXを固定する。
```

Layer 0が不安定であれば、監査対象そのものが不安定である。

---

## 5. Layer 1：動態遷移層

### 5.1 機能

論証はしばしば遷移を記述・含意する。  
それは静止画ではない。

Layer 1は、主張の背後にある動態構造を監査する。

### 5.2 最小項目

- 初期状態
- 入力
- 処理規則
- 中間状態
- 出力
- 分岐条件
- 更新条件
- 停止条件
- フィードバックループ

### 5.3 典型的失敗モード

- 結論だけ示され、過程がブラックボックス化される
- 一時的条件が恒久的ルールとして扱われる
- 分岐経路が消される
- 更新が無視される
- フィードバック効果が省略される

### 5.4 TCP写像

```text
Layer 1は主にRを固定する。
```

状態・関係・遷移がどう作動すると仮定されているかを明確にする。

---

## 6. Layer 2：暗黙知監査層

### 6.1 機能

Layer 2は、論証の背後にある隠れた支えを抽出する。

本フレームワークでは、4項目を中核監査対象とする。

```text
definition
premise
scope
uncertainty
```

これらは網羅ではない。  
本適用例で採用する最小中核項目である。

### 6.2 定義

監査質問：

- 重要語は明示的に定義されているか
- 途中で定義がずれていないか
- 別文脈の定義が暗黙に輸入されていないか

### 6.3 前提

監査質問：

- その主張が成立するために何を仮定しているか
- 前提は事実前提・制度前提・価値前提・比較前提・観測前提のどれか
- 必要前提と補助前提は区別されているか

### 6.4 射程

監査質問：

- 主張の有効範囲はどこか
- 普遍主張か条件付き主張か
- 領域限定か
- 非適用条件は示されているか

### 6.5 不確実性

監査質問：

- 事実・推論・仮説・予測・印象・願望のどれか
- 主張強度は根拠強度に見合っているか
- 不確実性が確実性に偽装されていないか

### 6.6 TCP写像

```text
Layer 2は主にMを固定する。
```

論証を解釈・判定するための閉包条件を与える。

---

## 7. Layer 3：明示論証監査層

### 7.1 機能

前段の層が固定された後、明示論証の質を評価する。

Layer 3では次の3項目を扱う。

```text
validity
coherence
adequacy
```

### 7.2 妥当性

推論形式は成立しているか。  
飛躍・循環・根拠なき結論はないか。

### 7.3 整合性

与えられた文脈内で記述が噛み合っているか。

整合性は文脈依存である。  
孤立文の無矛盾だけではない。

### 7.4 十分性

前提・根拠・観測・因果仮定・比較は、結論の強さに対して十分か。

形式的妥当性があっても、前提が腐っていれば論証は失敗する。

### 7.5 TCP写像

```text
Layer 3は、Layer 2で抽出されたM条件の下でRを監査する。
```

---

## 8. Layer 4：統合判定層

### 8.1 機能

Layer 4は層横断比較を行う。

次を横断したとき、論証が安定しているかを問う。

- 座標
- 動態
- 暗黙知
- 明示命題構造

### 8.2 一貫性

本フレームワークでは、整合性と一貫性を分ける。

```text
整合性 = 与えられた文脈内での局所的な噛み合い
一貫性 = 層・文脈をまたいだ支配原理の安定性
```

### 8.3 失敗類型

典型的な一貫性失敗は次である。

- 定義ずらし型
- 前提すり替え型
- 射程の拡大・縮小型
- 不確実性偽装型
- 無許可の座標移動型
- 動態省略型

### 8.4 TCP写像

```text
Layer 4は、X / R / Mを横断してWが安定しているかを確認する。
```

---

## 9. Layer 5：予約 / SUSPEND層

### 9.1 機能

Layer 5は、本フレームワークが完成理論を装うことを防ぐ。

ここに置かれるもの：

- 認識しているが展開していない項目
- まだ操作化されていない項目
- 現在の射程外の項目
- 中核構造を拡散させる項目
- 将来拡張候補

### 9.2 SUSPEND

座標・動態・暗黙支え・判定規則が未閉包なら、正しい出力は過剰断定ではない。

正しい出力は：

```text
SUSPEND
```

### 9.3 TCP写像

```text
Layer 5は、TCP的停止可能性の運用表現である。
```

## 10. 標準監査手順

1. **座標固定**  
   Who / What / When / Where / Why / How を固定する。

2. **動態抽出**  
   初期状態、入力、遷移、分岐、更新、停止、フィードバックを抽出する。

3. **暗黙知抽出**  
   定義、前提、射程、不確実性を抽出する。

4. **暗黙知評価**  
   抽出された暗黙要素が座標・動態と噛み合うか確認する。

5. **明示論証評価**  
   妥当性、整合性、十分性を確認する。

6. **統合判定**  
   層を横断して比較し、一貫性失敗を特定する。

7. **予約開示**  
   現監査の外側に残るものと、SUSPENDが必要かを明示する。

## 11. 例：「AIは世界を変える」

次の文：

```text
AIは世界を変える。
```

これはスローガンとしては意味を持ち得る。  
しかし、まだ監査可能な主張ではない。

### 11.1 座標失敗

- 誰が語っているのか
- AIとは何か
- 世界とは何か
- いつ変わるのか
- どこで変わるのか
- なぜその主張が重要なのか
- どのように変化が起きるのか

### 11.2 行為主体の曖昧性

行為主体が不明である。

- AIが自己変容するのか
- AIが直接何かを変えるのか
- 人間がAIを使うことで行動を変えるのか
- 組織がAIを導入することで制度が変わるのか
- AIがインセンティブを変えることで市場が変わるのか

### 11.3 射程の曖昧性

「世界」は次を意味し得る。

- 技術
- 経済
- 社会制度
- 人間認知
- 関係性
- 権力構造
- 情報環境

これらを一つの未分化な対象として監査することはできない。

### 11.4 動態の曖昧性

メカニズムが欠けている。

- 導入
- 利用
- 運用
- 制度変化
- 行動変化
- 権力移動
- フィードバック
- 規制
- 失敗と訂正

### 11.5 監査結果

```text
Decision: SUSPEND
Reason: 座標条件および動態条件が十分に固定されていない。
```

この文はレトリックとしては機能し得る。  
しかし、監査可能な命題としてはまだ成立していない。

---

## 12. 例：形式的には妥当だが監査に失敗する文

主張：

```text
このシステムは効率を上げる。
効率を上げるシステムは望ましい。
したがって、このシステムは望ましい。
```

形式上は妥当に見える。

しかし、次が固定されていなければ監査に失敗する。

- 誰にとって効率的か
- 何のタスクにおいて効率的か
- どの時間軸で効率的か
- どの制度空間で効率的か
- 副作用は含めるのか
- なぜ効率向上が望ましさに接続するのか
- 効率の定義は安定しているか
- 射程はどこか
- 不確実性はどう扱うか

判定例：

```text
Decision: SUSPEND or FAIL
Reason: 暗黙前提と射程が未定義である。
```

---

## 13. 例：局所整合性はあるが一貫性がない場合

文脈A：

```text
確率的出力システムに最終決定権を与えるべきではない。
```

文脈B：

```text
この確率的システムは高性能なので、最終決定権を委ねてもよい。
```

各文脈だけを見れば、局所的な整合性はあり得る。

しかし、文脈Bで性能が確率性を上書きする理由が説明されなければ、支配原理は不安定である。

判定例：

```text
Decision: FAIL or SUSPEND
Reason: 文脈横断での支配原理が一貫していない。
```

## 15. 意義

本フレームワークが有用なのは、次の理由による。

1. 論証監査を命題から成立条件へ拡張する
2. 時間・空間・動態を監査に導入する
3. 整合性と一貫性を分離する
4. 暗黙知を抽出可能にする
5. 圧縮モデルが閉じた全体理論を装うことを防ぐ
6. TCPの実用的監査ユースケースを与える

## 16. 限界

### 16.1 監査者依存

座標固定と暗黙知抽出には、監査者の能力が必要である。

### 16.2 操作化ギャップ

今後、各層を以下へ落とす必要がある。

- 質問票
- チェックリスト
- reason code
- 判定表
- テストケース

### 16.3 実証状態

これは概念的・方法論的フレームワークである。  
現時点では、経験的に検証された監査ベンチマークではない。

### 16.4 自己射程の制限

本フレームワーク自体も開放系である。  
完成された全体理論として扱ってはならない。
