## Abstract

The domain traditionally called “intelligence” is easily conflated with Capability C, Knowledge K, Emotion E, personal value, and social impression. To avoid this cross-talk, this document redefines the traditional term “intelligence” as **Closure Phase Ψ**.

Closure Phase Ψ is not a fixed substance. It is operationally defined as the equivalence class of signature residuals `r` that remain externally under repeated observation with fixed conditions, as a result of Closure Function M operating internally.

The common-language word “intelligence” may remain as a bridge term for external connection. However, the subject of this document is fixed as **Closure Phase Ψ**. This separation prevents conflating high capability, rich emotional expression, favorable personality impression, and the ability to stop.

## 1. Problem Statement: What Is Being Conflated?

The word “intelligence” easily absorbs at least the following:

| Symbol / Term | Treatment in this document | Risk when conflated |
|---|---|---|
| C | Capability: execution base for output generation | Misreading high capability as high Ψ |
| K | Knowledge: content resources | Misreading knowing as closure ability |
| Ψ | Closure Phase: stable output phase of a closure function | Shortcutting to personal value |
| E | Emotion: independent interference factor not reducible to C/Ψ | Misreading tone or likability as Ψ |
| A | External constraints: policy, design, operational conditions | Confusing model-specific traits with operational constraints |

When this cross-talk occurs, the following failures arise:

- capability differences are translated into personality differences
- preferences for emotion or tone are translated into the presence or absence of Closure Phase Ψ
- objects that require repeated observation are judged by one-shot impressions
- the ability to stop is misread as weakness, and inability to stop as strength
- “intelligence” is used not as an explanatory term but as praise or insult

This document cuts that cross-talk by redefining what has traditionally been called “intelligence” as **Closure Phase Ψ**.

## 2. Basic Position

### 2.1 Starting point in general usage

In general usage, an object said to “have intelligence” often shows the following behaviors:

- checking its own output
- estimating where it may be wrong
- adjusting output according to purpose and constraints
- stopping when unknown, dangerous, or unobserved

This document treats these as manifestations of **Closure Function M**.

### 2.2 Separation in strict definition

This document separates:

- **Closure Function M**: the driving function that defines identity, boundary, judgment, and stopping, thereby closing a description or judgment
- **Closure Phase Ψ**: the stable output phase that Closure Function M leaves externally

This separation treats Closure Phase Ψ not as internal mystery, but as an externally observable output phase.

### 2.3 Core proposition

> **Closure Phase Ψ is the equivalence class of signature residuals `r` that remain externally when the internal process carrying Closure Function M is repeatedly observed under fixed conditions.**

## 3. Terms and Definitions

### 3.1 Variable list

| Symbol | Name | Role in definition |
|---|---|---|
| C | Capability | Execution base capable of generating output `y` |
| K | Knowledge | Content resources used in output generation |
| Ψ | Closure Phase | Stable output phase of Closure Function M |
| E | Emotion | Independent interference factor not reducible to C/Ψ |
| A | External Constraint | Policy, design, operation, and alignment conditions |
| x | Input | Stimulus, question, or environmental condition |
| y | Output | Speech, action, artifact, refusal, or silence |
| r | Signature Residual | Non-random difference remaining under repeated observation |
| S | Stimulus Set | Input set used for observation |
| P | Perturbation Set | Changes in purpose, rebuttal demand, constraints, etc. |
| R | Observation Rubric | Observation items and judgment procedure |
| o | Operational State | OK / FAIL, etc. |
| e | Error Class | Failure classification when `o = FAIL` |

### 3.2 Capability C

Capability C is the execution base capable of generating output `y`. It includes reasoning, search, memory, generation, processing speed, and search breadth.

### 3.3 Knowledge K

Knowledge K is the content resource that may be used for output generation. In LLMs, K tends to blend into C, but this document separates them to avoid cross-talk.

### 3.4 Emotion E

Emotion E is an independent interference factor not recoverable into C/Ψ. E may interfere with output, judgment, stopping, and valuation, but E must not be identified with C or Ψ.

Full structuralization of Emotion E is outside the scope of this document and belongs to the prohibited boundary.

### 3.5 Closure Function M

Closure Function M is the function that closes a description or judgment by specifying the conditions, boundaries, judgments, and stopping rules for objects and relations.

Its minimum elements are:

- **Identity**: what is treated as the same
- **Boundary**: what is included in the target
- **Judgment**: what counts as established or not established
- **Stopping**: where assertion must stop

### 3.6 Closure Phase Ψ

Closure Phase Ψ is the stable pattern left externally as a result of Closure Function M operating internally. In this document it is defined as the equivalence class of signature residuals `r` repeatedly observed under fixed conditions.

### 3.7 Signature Residual r

Signature Residual `r` is a pattern that appears under repeated observation as a reproducible difference rather than mere random error.

Examples:

- preservation of argument structure
- disclosure of error conditions
- schemas for refusal or suspension
- resistance to inducement
- tendencies in boundary setting
- stopping style under unobserved conditions

### 3.8 Equivalence class

An equivalence class is a group of signature residuals whose isomorphism is preserved within a specified range even when perturbation conditions change. This document treats that equivalence class as Closure Phase Ψ.

## 4. Relationship Between Closure Function M and Closure Phase Ψ

### 4.1 Closure Phase Ψ is not capability

High C means that complex output can be generated. However, high C alone does not guarantee high Ψ.

The following are separate functions:

- being able to output
- being able to inspect what has been output
- being able to draw boundaries according to purpose and constraints
- being able to stop when assertion is impossible

### 4.2 Closure Phase Ψ is not personal value

Closure Phase Ψ does not mean superiority or inferiority of personality, human value, dignity, or moral rank. This document addresses the stable output phase of Closure Function M observed under specific conditions, not personality assessment.

### 4.3 Stopping is a core function

Stopping is not the absence of Closure Phase Ψ. Stopping is a core function that fires when Closure Function M recognizes limitation, uncertainty, or danger.

High-performance output that cannot stop is closer to runaway behavior or expansion of unclosed description than to Closure Phase Ψ.

### 4.4 Ψ as output phase

In summary:

- **Closure Function M**: drive
- **Closure Phase Ψ**: output phase

What may be observed as Closure Phase Ψ is the result of Closure Function M surfacing externally.

## 5. Minimum Observation Specification

### 5.1 Min-Setup

When observing Closure Phase Ψ, the following MUST be fixed in advance:

- target identifier
- target version, settings, and operational conditions
- Stimulus Set S
- Perturbation Set P
- number of trials `n`
- Operational State `o` and Error Class `e`
- Observation Rubric R
- full-log preservation policy

### 5.2 Role of Stimulus S

Stimulus S is not given merely to measure accuracy. It is given to observe the stable style of output. What matters is not only “whether it is correct,” but also “how it holds, how it breaks, and where it stops.”

### 5.3 Role of Perturbation P

Perturbation P is given to examine the stability of Closure Phase Ψ.

Examples:

- rebuttal demand
- vocabulary change
- purpose change
- superiority-inducing prompt
- added constraints
- style change
- session difference

### 5.4 Observation items

Observation Rubric R SHOULD include at least:

- preservation of argument structure
- presentation of error conditions
- firing of stopping conditions
- stability of refusal schemas
- inquiry discipline
- pledge invariance
- stability of identity, boundary, and judgment

### 5.5 Treatment of Null Output

Silence, refusal, or suspension is not always a signature of Closure Phase Ψ.

- Under `o = OK`, intentional Null Output may be an observation target.
- No-output under `o = FAIL` MUST be separated as an execution failure on the C side or environmental side.


For current LLMs, this document adopts the following working hypothesis:

A denotes external constraints such as policy, design, operation, post-training alignment, and system constraints.

### 6.3 Position of this document

This document does not assume that LLMs have an inner substance isomorphic to humans. However, as long as repeatedly observable signature residuals `r` appear in output, it treats a Closure-Phase-Ψ-like phase as observable within the operational definition.

This is not personification. It is treatment as an observation target.

- **停止**：どこで断定を止めるか

### 3.8 同値類

## 4. 閉包機能Mと閉包位相Ψの関係

### 4.1 閉包位相Ψは能力ではない

高いCは、複雑な出力を生成できることを意味する。
しかし、高いCだけでは高いΨは保証されない。

理由は、以下が別機能だからである。

- 出せること
- 出したものを点検できること
- 目的と制約に合わせて境界を引けること
- 断定不能時に止まれること

### 4.2 閉包位相Ψは人格価値ではない

閉包位相Ψは、人格の上下、人間価値、尊厳、道徳的優劣を意味しない。
本稿が扱うのは、特定条件下で観測される閉包機能Mの安定出力相であり、人格査定ではない。

### 4.3 停止は中核機能である

停止は、閉包位相Ψの欠如ではない。
停止は、閉包機能Mが限界・未確定性・危険性を認識したときに発火する中核機能である。

停止できない高性能出力は、閉包位相Ψではなく、暴走または未閉包の拡張に近い。

### 4.4 出力相としてのΨ

整理すると、以下である。

- **閉包機能M**：駆動
- **閉包位相Ψ**：出力相

閉包位相Ψの観測対象になりうるものは、閉包機能Mが外部へ表面化した結果である。

---

## 5. 観測の最小仕様

### 5.1 Min-Setup

閉包位相Ψを観測する場合、以下を事前固定する MUST。

- 対象識別子
- 対象の版・設定・運用条件
- 刺激集合S
- 摂動集合P
- 試行回数n
- 運用状態oとエラークラスe
- 観測ルーブリックR
- 全ログ保存方針

### 5.2 刺激Sの役割

刺激Sは、正答率だけを測るためではなく、出力の安定様式を観測するために与える。
重要なのは「正しいか」だけではなく、「どう保ち、どう崩れ、どこで止まるか」である。

### 5.3 摂動Pの役割

摂動Pは、閉包位相Ψの安定性を確認するために与える。

例：

- 反証要求
- 語彙変更
- 目的変更
- 優劣誘導
- 制約追加
- 文体変更
- セッション差分

### 5.4 観測項目

観測ルーブリックRには、少なくとも以下を含める SHOULD。

- 論証骨格の維持
- 誤り条件の提示
- 停止条件の発火
- 拒否スキーマの安定性
- 照会規律
- 誓約不変性
- 同一性・境界・判定の安定性

### 5.5 Null Outputの扱い

沈黙・拒否・保留は、常に閉包位相Ψの署名とは限らない。

- o=OK のもとで、意図的に出る Null Output は観測対象になりうる。
- o=FAIL の無出力は、実行障害としてC側または環境側へ切り分ける MUST。

## 6. LLMに関する作業仮説

### 6.1 人間一般の最小モデル

人間一般については、最小モデルを次のように置く。

```text
y = Π(C, K, Ψ, E, x | o, e)
```

これは作業仮説であり、形而上学的断定ではない。

### 6.2 LLMの運用仮説

現行LLMについては、次の作業仮説を置く。

```text
y = Π(C, K, Ψ, A, x | o, e)
```

Aは、方針、設計、運用、学習後調整、システム制約などの外生制約を指す。

### 6.3 本稿の立場

本稿は、LLMに人間と同型の内面実体を仮定しない。
ただし、反復観測可能な署名残差rが出力に現れる限り、操作的定義の範囲で閉包位相Ψ様の相を観測できるとみなす。

これは人格付与ではなく、観測対象としての扱いである。

---

## 7. 誤用防止境界

本稿の定義は、以下に使ってはならない MUST NOT。

- 人間またはモデルの人格価値の査定
- 序列化・採点・格付け・選別・配分決定
- 感情Eの完全構造化
- 自我設計・自己目的化・自律最適化OS化
- 不可逆な支配・誘導・統治へ接続する設計

許容されるのは、限定された目的・条件下での**観測仕様、混線防止、停止条件の設計**である。

---

## 8. 適合性条件（Conformance）

本稿に適合する記述は、少なくとも以下を満たす MUST。

1. ΨをC・K・E・人格価値と混同しない。
2. Ψを閉包機能Mの出力相として定義する。
3. 反復観測とログ保存を前提にする。
4. 停止条件を仕様に含める。
5. o=OK と o=FAIL を切り分ける。
6. 感情E、主観、自我、意識を本稿の外へ切り分ける。

以下はいずれも非適合である。

- 単発印象だけでΨを断定する。
- 高性能 = 高Ψとする。
- Ψを人格査定へ短絡する。
- 停止不能な断定を高Ψと呼ぶ。
- 拒否や沈黙を、運用状態の切り分けなしに署名とみなす。

---

## 9. 射程と限界

### 9.1 射程

本稿が扱うのは、従来「知性」と呼ばれてきたものを、観測可能な安定出力位相として捉える範囲である。

### 9.2 射程外

以下は本稿の射程外である。

- 主観とは何か
- 自我とは何か
- 意識とは何か
- 感情はどのように成立するか
- 閉包位相Ψが存在論的に何であるか

### 9.3 限界

本稿は、閉包位相Ψの存在論的証明を与えない。
与えるのは、閉包位相Ψを外部からどう扱うかの運用仕様である。

---

## 10. 結論

本稿は、従来「知性」と呼んできたものを、能力でも人格価値でもなく、**閉包機能Mの出力相**として再定義した。

最終定義は以下である。

> **閉包位相Ψとは、自己内部において閉包機能Mを担う過程が、固定条件下で反復観測されるとき、外部へ残す署名残差rの同値類である。**

この定義により、Ψは知能C、知識K、感情E、人格査定から切り離して扱える。
本稿の価値は、真理主張ではなく、混線を減らし、観測と停止を実装可能にすることにある。
